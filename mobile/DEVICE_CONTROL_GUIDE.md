# 📱 Приложение для удаленного управления WatersysPro

## 🎯 Назначение приложения

Это мобильное приложение React Native позволяет:
- 📊 **Мониторить** состояние устройства в реальном времени
- 🎮 **Управлять** реле и параметрами устройства удаленно
- 📈 **Просматривать** исторические данные и графики
- 🔔 **Получать** push-уведомления о событиях
- 🔐 **Безопасно** подключаться через JWT токены

## 📡 API Endpoints

### Аутентификация
```
POST /api/auth/login
  { email, password }
  ✓ Возвращает JWT токен

POST /api/auth/register  
  { email, password }
  ✓ Регистрация нового пользователя
```

### Устройства
```
GET /api/devices
  ✓ Список всех доступных устройств

GET /api/status/{deviceId}
  ✓ Текущее состояние устройства
  ✓ Температура, давление, влажность и т.д.
```

### Управление
```
POST /api/command/{deviceId}
  { command: "relay_on", relay: 1 }
  ✓ Отправить команду на устройство
```

### История и аналитика
```
GET /api/history/{deviceId}?days=7
  ✓ Исторические данные за 7 дней

GET /api/history-aggregate/{deviceId}?metric=pressure&days=7
  ✓ Агрегированные данные (средние значения)
```

### Push-уведомления
```
POST /api/push/register
  { token: "expo_push_token" }
  ✓ Регистрация для получения push-уведомлений
```

## 🔧 Конфигурация

### Подключение к серверу

В `src/services/api.ts` указан адрес сервера:
```typescript
const API_URL = 'http://192.168.3.13:80/api';
```

**Измените IP адрес если применимо:**
```typescript
// Для локального тестирования
const API_URL = 'http://192.168.1.100:3000/api';

// Для production
const API_URL = 'https://api.yourdomain.com/api';
```

### Безопасность

✅ Все запросы автоматически добавляют JWT токен в заголовок:
```
Authorization: Bearer <token>
```

✅ Временный лимит на запросы: 8 секунд

## 📲 Использование приложения

### 1. Первый запуск
```
1. Запустите приложение
2. Зарегистрируйтесь или авторизуйтесь
3. Разрешите push-уведомления
4. Перейдите на DashboardScreen
5. Начните управлять устройством
```

### 2. Мониторинг
- Просмотр текущих значений в реальном времени
- Прокрутка вниз для увеличения графиков
- Выбор периода для анализа (7, 30, 90 дней)

### 3. Управление
- Нажимайте кнопки для отправки команд
- Примеры команд:
  ```json
  { "command": "relay_on", "relay": 1 }
  { "command": "relay_off", "relay": 1 }
  { "command": "set_config", "param": "max_pressure", "value": 40 }
  ```

### 4. Уведомления
- Получайте оповещения о критических состояниях
- Уведомления приходят через Expo Push Notifications
- Все уведомления логируются в истории

## 🔐 Безопасность

### JWT Токены
- Автоматически добавляются во все запросы
- Хранятся в глобальной переменной (temp session)
- Проверяются на backend

### Шифрование
- HTTPS рекомендуется для production
- WebSocket может использовать WSS

### CORS
- Backend должен разрешать мобильное приложение
- Проверьте CORS headers в nginx.conf

## 🧪 Примеры использования API

### Пример 1: Получить статус
```typescript
import ApiService from './services/api';

const status = await ApiService.getStatus('device_001');
console.log(status);
// {
//   temperature: 23.5,
//   humidity: 45,
//   pressure: 35.2,
//   relay_1_state: 1,
//   ...
// }
```

### Пример 2: Отправить команду
```typescript
const response = await ApiService.sendCommand('device_001', {
  command: 'relay_on',
  relay: 1
});
```

### Пример 3: Получить историю
```typescript
const history = await ApiService.getHistory('device_001', 7); // 7 дней
// [
//   { timestamp: 1678...., temperature: 23.5, ... },
//   { timestamp: 1678...., temperature: 23.6, ... },
//   ...
// ]
```

### Пример 4: Подписаться на WebSocket события
```typescript
import { WebSocketService } from './services/websocket';

const ws = new WebSocketService('ws://192.168.3.13:8080');
ws.on('device_update', (data) => {
  console.log('Обновление:', data);
});
```

## 🆘 Troubleshooting

### "Cannot connect to device"
1. Проверьте IP адрес сервера в api.ts
2. Убедитесь что сервер запущен
3. Проверьте firewall правила
4. Перезапустите приложение (npm start)

### "Invalid token"
1. Выйдите из приложения
2. Авторизуйтесь заново
3. Проверьте backend логи

### "WebSocket disconnected"
1. Проверьте WebSocket URL
2. Убедитесь что backend поддерживает WebSocket
3. Проверьте proxy/firewall

### "Push notifications not working"
1. Проверьте что разрешены уведомления в ОС
2. Убедитесь что токен зарегистрирован на backend
3. Проверьте Expo Push Notification сервис
