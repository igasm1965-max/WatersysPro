# Синхронизация времени через NTP после подключения к WiFi

## 📋 Описание проблемы
После выключения питания модуль реального времени (RTC DS3231) может терять точность времени. Решением является синхронизация времени через интернет (NTP - Network Time Protocol) после успешного подключения к WiFi.

## ✅ Реализованное решение

### 1. **Функция синхронизации NTP** (`src/timers.cpp`)
```cpp
void syncTimeWithNTP()
```
- Использует встроенные функции ESP32 для получения времени через NTP
- Поддерживает 3 NTP сервера для отказоустойчивости: `pool.ntp.org`, `time.nist.gov`, `time.google.com`
- Ждет получения времени максимум 20 секунд
- При успешной синхронизации устанавливает время в RTC модуль DS3231
- Логирует результаты в событиях системы

### 2. **Автоматический вызов при подключении WiFi** (`src/wifi_manager.cpp`)
```cpp
WiFi.onEvent([](WiFiEvent_t event, WiFiEventInfo_t info){
  if (event == ARDUINO_EVENT_WIFI_STA_GOT_IP) {
    Serial.printf("[WiFi] STA got IP: %s\n", WiFi.localIP().toString().c_str());
    Serial.println("[WiFi] Запускаем синхронизацию времени по NTP...");
    syncTimeWithNTP();
  }
}, ARDUINO_EVENT_WIFI_STA_GOT_IP);
```

## 🔄 Процесс работы

1. **DeviceION подключается к WiFi**
   - ESP32 инициирует подключение к сохраненной сети

2. **Получен IP адрес**
   - Срабатывает событие `ARDUINO_EVENT_WIFI_STA_GOT_IP`
   - Обработчик события вызывает `syncTimeWithNTP()`

3. **Синхронизация времени**
   - ESP32 контактирует с NTP серверами
   - Получает точное текущее время по UTC+0
   - Устанавливает это время в модуль RTC DS3231

4. **Верификация**
   - Логирует успешную синхронизацию или ошибку в системные события
   - Выводит синхронизированное время в Serial порт

## 📊 Технические детали

### Используемые компоненты:
- **configTime()** - настройка NTP синхронизации
- **time()** - получение текущего времени в формате Unix timestamp
- **RTC_DS3231** - локальное сохранение синхронизированного времени
- **Event Logging** - отслеживание успеха/неудачи синхронизации

### Преимущества решения:
✅ Автоматическое исправление времени после перезагрузки
✅ Устойчивость к потере питания
✅ Не требует ручного вмешательства пользователя
✅ Использует встроенные функции ESP32 (нет доп. библиотек)
✅ Асинхронная работа без блокирования основного цикла
