## 🚀 Основные возможности

### 1. Поддержка MQTT TLS (Решена месячная проблема!)

**Решена проблема:**
- Старая библиотека (AsyncMqttClient v0.9.0) не имела поддержки TLS/SSL
- Отсутствующие методы: `setSecure()`, `setInsecure()`
- Полностью невозможно было исправить обновлением библиотеки

**Реализованное решение:**
- Заменено на PubSubClient v2.8.0 (стандартный Arduino MQTT клиент)
- Полная поддержка TLS/SSL через `NetworkClientSecure`
- Двойная конфигурация: plain (порт 19160) + TLS (порт 19161)
- Оба режима протестированы и проверены работающими

**Протестированная конфигурация:**
```
Брокер: m1.wqpt.ru
Учетные данные: REDACTED_MQTT_USER / REDACTED_MQTT_PASS
Режим plain: Порт 19160 ✅ Подключено
Режим TLS: Порт 19161 ✅ Подключено (TLS проверено)
Телеметрия: Публикует каждые 10s на watersystem/telemetry
```

### 2. Оптимизация памяти (Критическая проблема места Flash)

**Проблема:** Использование Flash 99.8% (только 12 KB свободно!) - не удалось скомпилировать новые функции

**Решение:**
- Изменена оптимизация с `-Os` на `-Oz` (агрессивное сокращение размера)
- Удалены исключения, RTTI, unwind таблицы
- Добавлены разделы функций/данных со сборкой мусора во время линковки
- Удалены отладочные символы

**Результат:** Flash сокращена с 99.8% → **94.6%** (освоболено 68 KB)
- Достигнут безопасный диапазон работы
- Место для будущих функций
- Та же функциональность, меньше места

## 📈 Технические детали

### Изменения прошивки

**Переработка MQTT:**
- Полная переработка vqtt.cpp (~400 строк)
- API PubSubClient (синхронный, проще чем асинхронный)
- Правильная синхронизация состояния WiFi
- ID устройства: `w_sys_XXXX_YYYYYYYY` (WiFi MAC + random)
- Экспоненциальная задержка: 5s → 60s макс между повторами

**Критическое исправление ошибки:**
- **Crash с нулевым указателем** когда WiFi не подключена при запуске
- Теперь инициализирует стандартный клиент безусловно
- Подключается только когда WiFi переходит в состояние подключено
- Предотвращает условие гонки при инициализации

**Хранилище конфигурации:**
- Новые ключи preferences: `PREF_KEY_MQTT_SECURE`, `PREF_KEY_MQTT_TLS_PORT`, `PREF_KEY_MQTT_INSECURE`
- Порт TLS по умолчанию: 8883
- Все настройки постоянны при перезагрузке

**Улучшения Web API:**
- `GET /api/mqtt` - извлечение всех настроек MQTT как JSON
- `POST /api/mqtt` - обновление конфигурации MQTT
- Включение/отключение TLS через веб-интерфейс
- Конфигурация порта TLS через веб-интерфейс

**Обновления инженерного меню:**
- Добавлено: переключатель TLS Enable/Disable
- Добавлено: редактирование портапорта TLS
- Интегрировано с существующей настройкой MQTT
- Очистка MQTT теперь удаляет настройки TLS

### Оптимизация компиляции

```ini
build_flags =
    -Oz                            # Агрессивный размер (vs -Os)
    -ffunction-sections            # Разделение на уровне функций
    -fdata-sections                # Разделение на уровне данных
    -fno-exceptions                # Удалить исключения
    -fno-rtti                      # Удалить RTTI
    -fno-unwind-tables             # Удалить таблицы разверток
    -fno-asynchronous-unwind-tables # Удалить асинхронные развертки
    -Wl,--gc-sections              # Сборка мусора при линковке
    -Wl,--strip-all                # Удалить все символы
```

## 🔄 Руководство миграции

### Для существующих развертываний

**Шаг 1: Обновить прошивку**
```bash
cd d:\WatersysPro
platformio run
platformio run -t upload
```

**Шаг 2: Конфигурировать MQTT (через веб-интерфейс)**
- Откройте http://192.168.3.13/
- Перейдите к настройкам MQTT
- Установите брокер: m1.wqpt.ru
- Установите порт: 19160 (plain) или 19161 (TLS)
- Установите учетные данные: REDACTED_MQTT_USER / REDACTED_MQTT_PASS
- Включите TLS: Да/Нет
- Сохраните конфигурацию

**Шаг 3: Проверить подключение**
- Проверьте серийный монитор: `[VQTT] ✓ Connected to broker!`
- Проверьте телеметрию: `[VQTT] Published telemetry... OK`

### Breaking Changes (Изменения с разрывом совместимости)

⚠️ **Важно**: Старый код AsyncMqttClient полностью заменен

- Библиотека изменена: `AsyncMqttClient@^0.9.0` → `PubSubClient@^2.8`
- API callback MQTT изменен (смотрите src/vqtt.cpp)
- Новые ключи preferences для конфигурации TLS
- Формат ID клиента изменен: `user_c7d5b9fb_XXXX` → `w_sys_XXXX_YYYYYYYY`

## 🐛 Известные проблемы и обходные пути

### Проблема: LTO Linker Errors
- **Симптом**: "plugin needed to handle lto object"
- **Статус**: ИСПРАВЛЕНО - Удно LTO, использование ручного удаления разделов вместо
- **Влияние**: Достигнута аналогичная экономия памяти

### Проблема: Flash слишком полна
- **Симптом**: "firmware too large"
- **Статус**: ИСПРАВЛЕНО - Освоболено 68 KB через оптимизацию
- **Влияние**: Теперь безопасно при 94.6% использовании

### Проблема: MQTT Null Pointer
- **Симптом**: Устройство crashes при запуске без WiFi
- **Статус**: ИСПРАВЛЕНО - Правильная синхронизация состояния
- **Влияние**: Стабильный запуск независимо от состояния WiFi
